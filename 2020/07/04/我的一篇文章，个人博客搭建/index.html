我的一篇文章，个人博客搭建
Created 2020-07-04| Updated 2020-07-04
Post View:

  • npm install -g cnpm –registry=http://registry.npm.taobao.org下载淘宝镜像

  • cnpm install -g hexo-cli\

    chdir查看文件

  • mkdir blog #创建blog目录

  • 下载git 重新打开cmd

  • chdir C:\Users\Administrator\blog hexo init #生成博客 初始化博客

  • hexo s #启动本地博客服务

  • hexo n “我的第一篇文章” #创建新的文章

  • hexo clean #清理

  • hexo g #生成

  • cnpm install –save hexo-deployer-git #在blog目录下安装git部署插件

  • _config.yml 文件修改

    type: ‘git’
    repo: https://github.com/qiujiangjiang/qiujiangjiang.github.io
    branch: master

  • 如果出现问题可能是用户名没有配置，打开git

hexo d 上传数据

  • git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia 博客主题下载

  • 经常报错curl 18 transfer closed with outstanding read data remaining

    • 将postBuffer值配置成合适的大小。

      git config –global http.postBuffer 524288000

      命令查看配置好的值postBuffer。

      git config –list

    • 如果还是不行

      ​ 直接在github下载源码放在themes主题中

      ​ _config.yml个修改一下

  • hexo c #清理一下
    hexo g #生成
    hexo d #部署到远程Github仓库

  • 如果出现fatal: HttpRequestException encountered. 发送请求时出错。问题

    Github 禁用了TLS v1.0 and v1.1，必须更新Windows的git凭证管理器，才行。

    https://github.com/microsoft/Git-Credential-Manager-for-Windows

cd 打开到你要管理的代码的文件夹后：

vi 文件名

esc zz 保存

git init 本地创建仓库

git add –all 文件添加到版本控制器

git commit -m “本次提交描述” 该命令会将git add .存入暂存区修改内容提交至本地仓库中，若文件未添加至暂存区，则提交时不会提交任何修改。

git remote add origin …(此处为git仓库地址) 本地文件和后台仓库关联

git push 上传代码

git push origin master 第一次上传代码的时候

git commit 提交代码到本地
