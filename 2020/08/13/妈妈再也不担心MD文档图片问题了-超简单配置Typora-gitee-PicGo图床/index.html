妈妈再也不担心MD文档图片问题了,超简单配置Typora+gitee+PicGo图床
发表于 2020-08-13| 更新于 2020-08-13|MarkDown
阅读量:

前言

1.1 环境:Windows10

1.2 所需软件:

  • Typora

  • Node.js

    • 官方下载地址:https://nodejs.org/en/

  • Gitee

    • 需要注册Gitee帐号
    • 官方地址:https://gitee.com/

  • PicGo

    • 官方下载地址：https://github.com/Molunerfinn/picgo/releases
    • macos下.dmg，window下.exe

搞快点

2.1 gitee新建仓库

  1. 点击头像边上+号,点击新建仓库

  2. 新建仓库需要注意选择公开以及使用readme文件初始化,最后选择确定,同时要记住仓库地址的后半截也就是归属+路径的部分需要用到

  3. 新建完仓库之后点击头像边上+号,点击设置

  4. 点击左侧私人令牌,选择projects并提交,之后会弹出令牌要保存好,只显示一次,如果遗失只能重新生成了!

2.2 PicGo配置

  1. 打开PicGo的界面,点击插件设置,在搜索栏中搜索gitee-uploader并安装

  2. 安装完成之后在左侧图床设置中找到gitee选项

    • repo这个位置就是仓库地址的后半截归属+路径
    • token这个位置就是填写私人令牌
    • 最后先点击确定再点击设为默认图床

2.3 Typora配置

  1. 点击Typora的菜单,左侧找到偏好设置

  2. 在左侧找到图像,插入图片时选择复制到指定路径,

    • 勾选对本地位置的图片应用上述规则以及对网络位置的图片应用上述规则

    • 在上传服务设定的上传服务选择PicGo,PicGo路径选择PicGo本地安装的路径

      • 全部完成可以点击验证图片上传选项

        正常时查看PicGO
